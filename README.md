# Хъсъл · Kaizen Planner v4

Личен 180-дневен планер (1 юли – 31 декември 2026): дневен dashboard, хранене, тренировки, контент план, навици, интерактивен календар и прогрес. Работи изцяло в браузъра и запазва offline-first поведението на оригиналната версия.

## Kaizen Edition

- Нова светла японска визуална система, вдъхновена от оригиналното лого: washi хартия, sumi-e мастило, enso кръг и един контролиран ember акцент.
- Адаптивен layout: фокусирана долна навигация на телефон и пълноценна странична навигация на по-широк екран.
- По-ясни дневен timeline, статистики, календар, навици и прогрес — без промяна на съществуващата логика и локални данни.
- Осем оптимизирани WebP food assets в `assets/meals/`. Всички 28 позиции от менюто получават подходящ кадър; сходните и повторени ястия преизползват един файл.
- Обновен PWA cache (`hustle-kaizen-v40`) с всички нови offline assets.

## Какво е новото във v2

- **Жив дизайн**: AI-генериран фон с тлееща жарава + анимирани искри, които се издигат по екрана. Колкото по-изпълнен е денят ти, толкова по-силно "гори" интерфейсът (пръстенът, логото и фонът реагират на прогреса).
- **Искри при всяка отметка** и експлозия от искри при 100% завършен ден.
- **Дневен dashboard** ("Днес"): всичко за деня на едно място — тема на седмицата, статистики с анимирани числа, пълният план час по час, твоите задачи, рефлексия.
- **Интерактивен календар**: месечен изглед с цветово кодиране (колко % от деня е изпълнен), точки за тренировка/видео/задачи, докосваш ден → отваря се.
- **Реални планове по дати**: тренировките и контент планът вече не са "ден 1–30 на цикъл", а конкретните дати от юлските ти файлове (Сила A на 6.07, Видео 1 на 5.07, Ден 30 отчетът на 30.07...).
- **Редакция от приложението**: бутон "Редактирай" в Хранене / Спорт / Контент — променяш ястие, тренировка или видео за конкретна дата без да пипаш код. Промените се пазят локално "върху" месечния план, с бутон "Оригинал" за връщане.
- **Месечни модули**: планът за всеки месец е отделен файл `data-YYYY-MM.js`. В края на месеца качваш новия файл и добавяш 1 ред в `index.html` — старите месеци остават за календара и статистиката.
- **Пазарски списък** с отметки (в таб Навици) — нулира се всяка седмица.
- **Тегло с графика** (в Прогрес) — записваш веднъж седмично, виждаш кривата.
- **Постижения (badges)**: 8 отключваеми значки — серии, тренировки, видеа, рефлексии.
- **Правилата за watch rate** от контент плана — винаги под ръка в таб Контент.

## Облачна синхронизация (телефон ↔ лаптоп)

От v3 приложението синхронизира всичко през Firebase Firestore: отметки, рефлексии, задачи, тегло, редакции. Локалният запис (localStorage) остава — офлайн всичко работи, а при връзка се слива автоматично с облака (по-новото печели, нищо не се губи).

**Еднократна настройка (5 минути):**

1. https://console.firebase.google.com → **Add project** → име напр. `zharava` → Google Analytics: изключи → Create.
2. В проекта: **Build → Firestore Database → Create database** → Location: `europe-west3 (Frankfurt)` → **Start in production mode** → Create.
3. Таб **Rules** → замени всичко с това и натисни **Publish**:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /zharava/state {
         allow read, write: if true;
       }
     }
   }
   ```
4. ⚙️ **Project settings → Your apps → </> (Web)** → име `zharava` → Register (без Hosting) → копирай обекта `firebaseConfig`.
5. Отвори `app.js`, намери блока `▼▼▼ ПОСТАВИ ТУК КОНФИГА` (почти в края на файла) и попълни стойностите (apiKey, projectId и т.н.).
6. Качи файловете в GitHub → отвори приложението на двете устройства → долу трябва да пише **„облак ✓"**.

Първото устройство, което се отвори, качва данните си в облака. Второто ги слива със своите. Оттам нататък всяка отметка се появява на другото устройство до 1–2 секунди.

> ⚠️ Правилата по-горе са отворени (както при работното ти приложение) — всеки, който знае projectId, теоретично може да чете/пише този документ. За личен планер с неотгатваемо име на проекта това е приемлив компромис без логин.

## Качване в GitHub Pages

1. Създай репозиторий (или ползвай съществуващия).
2. Качи всички файлове и папката `assets/meals/` със същата структура:
   ```
   index.html
   style.css
   app.js
   data-2026-07.js
   sw.js
   manifest.webmanifest
   logo-mark.png
   icon-192.png
   icon-512.png
   apple-touch-icon.png
   assets/meals/*.webp
   README.md
   ```
3. Settings → Pages → Deploy from a branch → main / (root) → Save.
4. Отвори `https://<потребител>.github.io/<repo>/`. Ако си обновявал старата версия — презареди с Ctrl+Shift+R (или затвори/отвори таба на телефона), за да падне старият кеш.
5. На телефона: меню → **Add to Home Screen** → приложението се инсталира с икона и работи офлайн.

## Как се сменя месецът (правиш го веднъж месечно)

1. Дай ми файловете за новия месец (тренировъчен план, меню, контент план) — аз ти връщам готов `data-2026-08.js`.
2. Качи файла в репозитория.
3. В `index.html` намери блока `===== МЕСЕЧНИ ПЛАНОВЕ =====` и разкоментирай/добави реда:
   ```html
   <script src="data-2026-08.js" defer></script>
   ```
4. В `sw.js` добави `"./data-2026-08.js",` в списъка ASSETS и увеличи версията на `CACHE` (например `hustle-kaizen-v40` → `hustle-kaizen-v41`).
5. Commit → готово. Календарът и прогресът виждат и стария, и новия месец.

## Данни

Всички отметки, рефлексии, тегло, редакции — локално в браузъра (localStorage). Нищо не се изпраща никъде.
- **Резервно копие**: Прогрес → Експорт (JSON).
- **Друго устройство**: там Прогрес → Импорт.
- Данните от старата версия (v1) се мигрират автоматично при първото отваряне.

## Структура

| Файл | Какво е |
|---|---|
| `index.html` | Скелет + списък на месечните модули |
| `style.css` | Адаптивната Kaizen визуална система |
| `app.js` | Логиката (vanilla JS, без библиотеки) |
| `data-2026-07.js` | Планът за юли: тренировки по дати, меню, контент, пазар, теми |
| `logo-mark.png` | Оригиналното лого — запазено без промяна |
| `assets/meals/*.webp` | Оптимизираните преизползваеми изображения на ястията |
| `sw.js` | Service worker — офлайн + мигновено отваряне |
