# Хъсъл · Live Planner v2

Личен 180-дневен планер (1 юли – 31 декември 2026): дневен dashboard, хранене, тренировки, контент план, навици, интерактивен календар и прогрес. Работи изцяло в браузъра — без сървър и без акаунти.

## Какво е новото във v2

- **Жив дизайн**: AI-генериран фон с тлееща жарава + анимирани искри, които се издигат по екрана. Колкото по-изпълнен е денят ти, толкова по-силно "гори" интерфейсът (пръстенът, логото и фонът реагират на прогреса).
- **Искри при всяка отметка** и експлозия от искри при 100% завършен ден.
- **Дневен dashboard** ("Днес"): всичко за деня на едно място — тема на седмицата, статистики с анимирани числа, пълният план час по час, твоите задачи, рефлексия.
- **Интерактивен календар**: месечен изглед с цветово кодиране (колко % от деня е изпълнен), точки за тренировка/видео/задачи, докосваш ден → отваря се.
- **Реални планове по дати**: тренировките и контент планът вече не са "ден 1–30 на цикъл", а конкретните дати от юлските ти файлове (Сила A на 6.07, Видео 1 на 5.07, Ден 30 отчетът на 30.07...).
- **Редакция от приложението**: бутон "Редактирай" в Хранене / Спорт / Контент — променяш ястие, тренировка или видео за конкретна дата без да пипаш код. Промените се пазят локално "върху" месечния план, с бутон "Оригинал" за връщане.
- **Месечни модули**: планът за всеки месец е отделен файл `data-YYYY-MM.js`. В края на месеца качваш новия файл и добавяш 1 ред в `index.html` — старите месеци остават за календара и статистиката.
- **Пазарски списък** с отметки (в таб Навици) — нулира се всяка седмица.
- **Тегло с графика** (в Прогрес) — записваш веднъж седмично, виждаш кривата.
- **Постижения (badges)**: 8 отключваеми значки — серии, тренировки, видеа, рефлексии.
- **Правилата за watch rate** от контент плана — винаги под ръка в таб Контент.

## Качване в GitHub Pages

1. Създай репозиторий (или ползвай съществуващия).
2. Качи ВСИЧКИ файлове от тази папка в корена на репозитория (тук няма подпапки — всичко е плоско, точно както обича GitHub uploader-ът):
   ```
   index.html
   style.css
   app.js
   data-2026-07.js
   bg-embers.webp
   sw.js
   firebase-config.js
   firebase-sync.js
   manifest.webmanifest
   icon.svg
   README.md
   ```
3. Settings → Pages → Deploy from a branch → main / (root) → Save.
4. Отвори `https://<потребител>.github.io/<repo>/`. Ако си обновявал старата версия — презареди с Ctrl+Shift+R (или затвори/отвори таба на телефона), за да падне старият кеш.
5. На телефона: меню → **Add to Home Screen** → приложението се инсталира с икона и работи офлайн.

## Как се сменя месецът (правиш го веднъж месечно)

1. Дай ми файловете за новия месец (тренировъчен план, меню, контент план) — аз ти връщам готов `data-2026-08.js`.
2. Качи файла в репозитория.
3. В `index.html` намери блока `===== МЕСЕЧНИ ПЛАНОВЕ =====` и разкоментирай/добави реда:
   ```html
   <script src="data-2026-08.js" defer></script>
   ```
4. В `sw.js` добави `"./data-2026-08.js",` в списъка ASSETS и смени `zharava-v3` на `zharava-v4` (за да се обнови кешът).
5. Commit → готово. Календарът и прогресът виждат и стария, и новия месец.

## Данни

Всички отметки, рефлексии, тегло, редакции — локално в браузъра (localStorage). Ако включиш Firebase (виж по-долу), същите данни се синхронизират и в облака между устройствата ти.
- **Резервно копие**: Прогрес → Експорт (JSON).
- **Друго устройство**: Firebase синхронизация (автоматично) или Прогрес → Импорт (ръчно).
- Данните от старата версия (v1) се мигрират автоматично при първото отваряне.

## Firebase · облачна синхронизация (по избор, безплатно)

С Firebase всяка отметка се пази в облака и се появява на всичките ти устройства (телефон + компютър). Без настройка приложението работи както досега — само локално.

### 1. Създай Firebase проект
1. Отвори [console.firebase.google.com](https://console.firebase.google.com) → **Add project** → име напр. `zharava` → Google Analytics не е нужен → **Create**.
2. От началния екран на проекта: иконата **`</>`** (Web) → име на приложението напр. `zharava-web` → **Register app** (без Firebase Hosting).
3. Ще видиш блок `firebaseConfig = { apiKey: "...", ... }` — **копирай стойностите**.

### 2. Попълни `firebase-config.js`
Отвори файла `firebase-config.js` в репозитория и попълни стойностите от стъпка 1:
```js
window.ZHARAVA_FIREBASE_CONFIG = {
  apiKey: "AIza...",
  authDomain: "zharava.firebaseapp.com",
  projectId: "zharava",
  storageBucket: "zharava.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abc123",
};
```

### 3. Включи входа с Google
1. Firebase Console → **Build → Authentication → Get started**.
2. Таб **Sign-in method** → **Google** → Enable → избери support email → **Save**.
3. Таб **Settings → Authorized domains** → **Add domain** → добави домейна на GitHub Pages: `<потребител>.github.io`.

### 4. Включи Firestore (базата данни)
1. Firebase Console → **Build → Firestore Database → Create database** → **Start in production mode** → регион напр. `europe-west3` → **Enable**.
2. Таб **Rules** → замени правилата с тези (всеки вижда само своите данни) → **Publish**:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /planners/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
  }
}
```

### 5. Влез в приложението
Commit + push на попълнения `firebase-config.js`, отвори приложението → таб **Прогрес → Данни → Вход с Google**. Готово — статусът горе вдясно става „в облака ✓“.

**Как работи синхронизацията:** локалното копие винаги е първо (офлайн всичко работи), облакът се обновява при всяка промяна. При отваряне на второ устройство печели по-новото състояние (по времеви печат). Данните са един документ `planners/{твоя-uid}` — безплатният план на Firebase стига в пъти.

## Структура

| Файл | Какво е |
|---|---|
| `index.html` | Скелет + списък на месечните модули |
| `style.css` | Всички стилове |
| `app.js` | Логиката (vanilla JS, без библиотеки) |
| `data-2026-07.js` | Планът за юли: тренировки по дати, меню, контент, пазар, теми |
| `bg-embers.webp` | AI фонът с жаравата |
| `sw.js` | Service worker — офлайн + мигновено отваряне |
| `firebase-config.js` | Твоите Firebase ключове (празен = без облак) |
| `firebase-sync.js` | Облачната синхронизация (Auth + Firestore) |
